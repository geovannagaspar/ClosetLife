1. Introdução

O ClosetLife é uma aplicação web desenvolvida com o objetivo de auxiliar usuários no gerenciamento digital de roupas, organização do guarda-roupa e controle visual de peças pessoais. O sistema foi criado aplicando conceitos fundamentais de Engenharia e Qualidade de Software, utilizando práticas relacionadas à modelagem, arquitetura em camadas, responsividade, usabilidade e organização modular do código. A proposta do projeto consiste em oferecer uma solução simples, moderna e intuitiva para facilitar a administração de roupas, favoritos, agenda pessoal e controle financeiro relacionado ao closet.

Público-alvo: Usuários que desejam organizar roupas, montar combinações e visualizar informações do closet de forma digital e acessível.

Escopo do Projeto: O sistema permite cadastro, visualização e gerenciamento de peças de roupas, além de recursos complementares como favoritos, agenda e controle financeiro, utilizando armazenamento local e interface responsiva.

2. Objetivos do Sistema

  • Desenvolver um sistema web funcional para gerenciamento de roupas
  • Aplicar conceitos de Engenharia e Qualidade de Software

  • Implementar interface responsiva para múltiplos dispositivos

5. Design e Identidade Visual

O design do ClosetLife foi planejado para transmitir elegância, leveza e organização. A paleta de cores utiliza tons neutros e pastéis, focando em uma estética minimalista que não compete com as imagens das roupas do usuário.

Elemento Especificação
Paleta de Cores Bege (#FDFBF8), Dourado Suave (#C8A97E), Cinza Escuro (#333)
Tipografia Montserrat (Google Fonts) - Foco em legibilidade e modernidade
Iconografia Feather Icons - Traços finos e minimalistas
Componentes Cards com bordas arredondadas e sombras suaves (Neomorfismo leve)

A experiência do usuário (UX) foi priorizada através de feedbacks visuais, como o uso do SweetAlert2 para confirmações de ações e estados de sucesso/erro.

6. Arquitetura do Sistema

O ClosetLife utiliza arquitetura em camadas, separando interface visual, lógica de negócio e persistência de dados. Essa organização facilita manutenção, escalabilidade e reutilização do código, além de melhorar a estrutura geral do sistema.

Camada Tecnologia
Interface HTML5 e CSS3
Lógica JavaScript (ES6+)
Persistência LocalStorage API e Canvas API (Processamento de Imagem)

7. Paradigmas de Desenvolvimento

O sistema utiliza conceitos de Programação Orientada a Objetos (POO) para organização das estruturas e componentes do projeto, além de práticas funcionais aplicadas em funções JavaScript responsáveis pela manipulação de dados e atualização dinâmica da interface.
