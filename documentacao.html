Documentação Técnica — ClosetLife

Engenharia e Qualidade de Software

1. Introdução

O ClosetLife é um sistema web desenvolvido para auxiliar usuários na organização digital de roupas, favoritos, agenda pessoal e controle financeiro relacionado ao closet.

O projeto foi desenvolvido aplicando conceitos de Engenharia e Qualidade de Software, priorizando organização visual, arquitetura modular, experiência do usuário, responsividade e facilidade de manutenção.

O sistema busca oferecer praticidade no gerenciamento do closet pessoal através de uma interface moderna, intuitiva e acessível.

2. Objetivo Geral

Desenvolver uma aplicação web responsiva para gerenciamento digital de roupas, favoritos, agenda e controle financeiro pessoal.

3. Objetivos Específicos

  • Aplicar conceitos de Engenharia de Software
  • Desenvolver sistema responsivo
  • Utilizar arquitetura em camadas
  • Aplicar Programação Orientada a Objetos
  • Utilizar armazenamento local com LocalStorage
  • Criar documentação técnica e UML
  • Implementar interface moderna e intuitiva
  • Melhorar experiência visual do usuário

4. Público-Alvo

O sistema é destinado a usuários que desejam organizar digitalmente roupas, favoritos, compromissos e informações relacionadas ao closet pessoal.

5. Escopo do Projeto

O ClosetLife permite:

  • cadastro de roupas;
  • upload de imagens;
  • gerenciamento de favoritos;
  • organização da agenda;
  • controle financeiro básico;
  • navegação entre páginas;
  • armazenamento local dos dados.

O sistema foi desenvolvido para ambiente web, funcionando em navegadores modernos e dispositivos móveis.

6. Aspectos Éticos e Segurança da Informação

Mesmo sendo um sistema acadêmico, o ClosetLife aplica conceitos básicos de segurança e organização de dados.

Recursos Aplicados:

  • Persistência local através do LocalStorage
  • Estrutura modular do sistema
  • Organização dos dados em JSON
  • Interface responsiva
  • Separação entre estrutura, estilo e lógica

7. Fundamentos e Objetivos da Arquitetura

A arquitetura do ClosetLife foi planejada para garantir:

  • organização modular;
  • facilidade de manutenção;
  • separação de responsabilidades;
  • escalabilidade futura;
  • reutilização de código.

Arquitetura em Camadas

Camada Tecnologia Responsabilidade
Apresentação HTML5 + CSS3 Interface visual
Lógica JavaScript ES6+ Processamento das funcionalidades
Persistência LocalStorage API Armazenamento dos dados
Navegação DOM Atualização dinâmica

Objetivo do Desacoplamento

A separação das camadas permite futuras alterações sem comprometer toda a aplicação. Exemplos: alterar o design não afeta a lógica; alterar o armazenamento não afeta a interface.

8. Princípios Arquiteturais

Separation of Concerns (SoC): Cada módulo possui responsabilidade específica.

DRY (Don't Repeat Yourself): Reutilização de código para evitar duplicações.

KISS (Keep It Simple): Estrutura simples, organizada e de fácil manutenção.

Responsividade: Compatibilidade com diferentes dispositivos e tamanhos de tela.

9. Paradigmas de Desenvolvimento

Programação Orientada a Objetos (POO)

Classe Responsabilidade Atributos
Roupa Gerenciar peças nome, categoria, imagem
Agenda Gerenciar eventos data, horário, evento
Financeiro Gerenciar valores valor, categoria, descrição

Benefícios: Melhor organização, facilidade de manutenção e escalabilidade.

Programação Funcional

O projeto utiliza manipulação de arrays, funções reutilizáveis, processamento de listas e atualização dinâmica da interface.
